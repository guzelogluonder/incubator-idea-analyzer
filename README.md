## ðŸ“Š Technical Decisions and Architecture

### Why Groq API?

- **High Performance**: Low latency, fast response times
- **Cost Effective**: More affordable pricing compared to OpenAI
- **OpenAI Compatible**: Same API format, easy migration
- **Llama 3**: Open source, powerful model

### Fallback Mechanism

- **Reliability**: System continues to work even if AI fails
- **Transparency**: Method used is marked in each analysis
- **Flexibility**: Easy control with environment variables

### Role Management Approach

- **Simplicity**: UX demonstration without login system complexity
- **LocalStorage**: Role persists across page refreshes
- **Context API**: State management with React best practices
